**Typography Migration Mapping Table:**

Include a section titled "Typography Migration" with this mapping for Plans 02-04 to follow:

| Raw Tailwind Classes | Semantic Token | Notes |
|---|---|---|
| `text-2xl font-bold` or `text-[28px] font-bold` | `text-h1` | Page titles |
| `text-xl font-semibold` or `text-[22px] font-semibold` | `text-h2` | Section headings |
| `text-lg font-semibold` or `text-[18px] font-semibold` | `text-h3` | Card titles, subsections |
| `text-base font-semibold` | `text-h4` | Labels, small headings |
| `text-sm` (body content) | `text-body` | Default body text (14px) |
| `text-xs` (captions, hints) | `text-caption` | Small helper text (12px) |
| `text-xs font-semibold uppercase tracking-wider` | `text-label uppercase tracking-wider` | UI labels (12px/600) |

Note: `text-h1` through `text-label` are composite tokens that set font-size, line-height, AND font-weight simultaneously. When migrating `text-xl font-bold` to `text-h2`, remove the separate `font-bold` class since `text-h2` includes `fontWeight: 600`.

Judgment calls: When a `text-sm` is clearly a label (short, descriptive) use `text-label`. When it's body content, use `text-body`. When `text-xs` appears on a substantive line (not a caption), keep `text-caption` — the scale treats 12px as the minimum readable size.

**Spacing Migration Note:**

Include a section titled "Spacing Approach" documenting:

Tailwind's default numeric spacing maps cleanly to our 4px grid:
- `p-1` = 4px = xs
- `p-2` = 8px = sm
- `p-3` = 12px = md
- `p-4` = 16px = lg
- `p-6` = 24px = xl
- `p-8` = 32px = 2xl
- `p-12` = 48px = 3xl

These numeric values ARE on-scale and do NOT need renaming. The semantic aliases (`p-xs`, `p-sm-space`, etc.) are available for readability but the numeric equivalents are equally valid.

Off-scale values that MUST be audited and corrected:
- `p-2.5` (10px) — not on the 4/8/12/16 grid. Replace with `p-2` (8px) or `p-3` (12px)
- `p-3.5` (14px) — not on grid. Replace with `p-3` (12px) or `p-4` (16px)
- `p-5` (20px) — not on grid. Replace with `p-4` (16px) or `p-6` (24px)
- `p-7` (28px) — not on grid. Replace with `p-6` (24px) or `p-8` (32px)
- `p-9` (36px) — not on grid. Replace with `p-8` (32px) or `p-10` (40px, still off) or `p-12` (48px)
- `p-10` (40px) — not on grid. Replace with `p-8` (32px) or `p-12` (48px)
- `p-11` (44px) — not on grid. Replace with `p-12` (48px)
- `p-14` (56px) — not on grid. Replace with `p-12` (48px)
- `p-0.5` (2px) — below minimum. Acceptable for micro-adjustments (icon alignment)
- `p-1.5` (6px) — not on grid. Replace with `p-1` (4px) or `p-2` (8px)

The same applies to `m-`, `px-`, `py-`, `mx-`, `my-`, `mt-`, `mb-`, `ml-`, `mr-`, `gap-`, `space-x-`, `space-y-`.

Context for replacements: Use visual judgment. `p-5` on a container is close enough to `p-6` (24px). `p-7` on a section padding can become `p-6` or `p-8` depending on density preference.
