In each file, perform THREE categories of replacement:

**A. Color replacements (dark mode surface and text):**

Background:
- `dark:bg-slate-900` -> `dark:bg-surface-dark-primary`
- `dark:bg-slate-800` -> `dark:bg-surface-dark-secondary`
- `dark:bg-slate-700` -> `dark:bg-surface-dark-elevated`
- `dark:bg-gray-900` -> `dark:bg-surface-dark-primary`
- `dark:bg-gray-800` -> `dark:bg-surface-dark-secondary`
- `bg-white` (page/card bg) -> `bg-surface-primary`
- `bg-gray-50` or `bg-neutral-bg` -> `bg-surface-secondary`

Text:
- `dark:text-white` -> `dark:text-text-theme-dark-primary`
- `dark:text-slate-100` or `dark:text-slate-200` -> `dark:text-text-theme-dark-primary`
- `dark:text-slate-300` or `dark:text-slate-400` -> `dark:text-text-theme-dark-secondary`
- `dark:text-slate-500` -> `dark:text-text-theme-dark-muted`
- `text-gray-900` or `text-neutral-body` -> `text-text-theme-primary`
- `text-gray-600` or `text-gray-500` -> `text-text-theme-secondary`
- `text-gray-400` -> `text-text-theme-muted`

Border:
- `dark:border-slate-700` or `dark:border-slate-600` -> `dark:border-surface-dark-elevated`
- `border-gray-200` or `border-gray-100` -> `border-surface-secondary`

Shadow:
- `shadow-sm` or `shadow` on cards -> `shadow-card dark:shadow-card-dark`
- `shadow-lg` or `shadow-xl` on modals -> `shadow-modal dark:shadow-modal-dark`

**B. isDark ternary patterns (Header.tsx, Footer.tsx):**

These files use `isDark ? 'class-a' : 'class-b'` string interpolation which bypasses the `dark:` prefix pattern.

- Header.tsx: `isDark ? 'bg-brand-primary' : 'bg-gray-200'` — replace `bg-gray-200` with `bg-surface-secondary`
- Footer.tsx: `isDark ? 'text-slate-600' : 'text-slate-400'` — replace with `isDark ? 'text-text-theme-dark-muted' : 'text-text-theme-muted'`

Search each file for the pattern `isDark ?` and inspect every occurrence. Replace raw slate/gray values in these ternaries with semantic token equivalents. Keep ternaries that reference brand-primary or other intentional non-theme colors.

**C. Typography migration (per design-system.md mapping table):**

Apply to all 6 files:
- `text-2xl font-bold` (headings) -> `text-h1` (remove the separate font-bold)
- `text-xl font-semibold` or `text-xl font-bold` (section heads) -> `text-h2` (remove separate font-weight class)
- `text-lg font-semibold` (card titles) -> `text-h3` (remove separate font-weight class)
- `text-base font-semibold` (small headings) -> `text-h4` (remove separate font-weight class)
- `text-sm` (body content, descriptions) -> `text-body`
- `text-xs` (captions, hints, helper text) -> `text-caption`
- `text-xs font-semibold uppercase` (UI labels) -> `text-label uppercase`

IMPORTANT: The semantic fontSize tokens (text-h1, text-body, etc.) include fontWeight. When migrating, REMOVE the separate font-bold/font-semibold/font-medium class that accompanied the old text-size class. Example: `text-xl font-bold` becomes just `text-h2` (not `text-h2 font-bold`).

Exception: If a `font-bold` or `font-semibold` appears WITHOUT an accompanying text-size (e.g., on a `<strong>` tag or inline emphasis), leave it as-is.

Exception: If `text-sm` or `text-xs` is used in a context where none of the semantic tokens fit (e.g., a non-standard size for a specific micro-component), use best judgment.

**D. Off-scale spacing audit:**

In each file, search for these off-scale spacing patterns and replace:
- `p-2.5` -> `p-2` (8px) or `p-3` (12px) depending on visual fit
- `p-5` -> `p-4` (16px) or `p-6` (24px) depending on context
- `p-7` -> `p-6` (24px) or `p-8` (32px)
- `p-0.5` -> acceptable for micro-alignment, leave as-is
- `p-1.5` -> `p-1` (4px) or `p-2` (8px)

Apply same logic to `m-`, `px-`, `py-`, `mx-`, `my-`, `mt-`, `mb-`, `ml-`, `mr-`, `gap-`, `space-x-`, `space-y-` variants.

**Important rules:**
- Do NOT change any functional logic, state management, or component structure
- Preserve ALL existing animations and transitions
- If a color is used for a specific non-theme purpose (e.g., brand-primary for accent), leave it as-is
