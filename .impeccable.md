## Design Context

### Users
Students (likely young adults, teens) at an English-language school in Uzbekistan. They access the platform to track lessons, submit assignments, take quizzes, and compete on leaderboards. Context is mobile-first — they're checking scores, starting quizzes, and seeing how they rank against classmates. Motivation is both academic (pass the quiz) and social/competitive (earn coins, climb the leaderboard).

### Brand Personality
Energetic · Competitive · Rewarding. The product is gamified — coins, rankings, XP. It should feel motivating, not clinical. The brand color (`#12c2dc`, a vivid teal-cyan) is the one distinctive asset; everything else should support it without overwhelming.

### Aesthetic Direction
Clean and functional with moments of energy. Not a sterile SaaS dashboard. Not a neon gaming UI. The sweet spot is a well-designed student app — think Duolingo's focus on clarity and feedback loops, but less cartoon-y. Light mode is primary. Dark mode is supported. Both must feel intentional.

### Design Principles
1. **Clarity over decoration** — Every visual element must earn its place. Gradients, shadows, and colored borders should be absent unless they carry meaning.
2. **Hierarchy by contrast, not quantity** — Use 2–3 type sizes max per component. Contrast between levels should be obvious, not subtle.
3. **Gamification through feedback** — The UI should make progress feel visible and rewarding. Scores, streaks, pass/fail — these are emotional moments. Treat them that way.
4. **Mobile-first density** — Cards and rows must breathe but not waste space. Students are checking on phones between classes.
5. **Coherent palette** — Brand-primary is the one accent. Semantic colors (green = pass, red = fail, amber = warning) should be used sparingly and only when communicating status.
